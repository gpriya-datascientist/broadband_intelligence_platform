## My Findings (Researcher Perspective)

### What is Promising for Our Work (Programming Task Generation + Educational Benchmarking)

| Finding | Evidence | Implication |
|---------|----------|-------------|
| GPT-4 is the most capable model across all task types tested | Papers 1, 8, 9, 10, 13, 21 | Strong baseline for task generation; use GPT-4 as reference model |
| Prompt engineering (CoT, augmented prompting) significantly improves output quality | Papers 2, 8, 20 | Task generation prompts must be carefully designed with constraints and feedback loops |
| LLMs can generate programming questions indistinguishable from human-written ones | Paper 5 | Confirms feasibility of LLM-based task generation; raises integrity concerns to address |
| Open-source models (LLaMA, Mistral, DeepSeek) are competitive but less reliable | Papers 1, 7, 9 | Should be included in benchmark comparison for reproducibility |
| CS1 performance is high; CS2 and advanced performance drops sharply | Papers 18, 23, 24 | Our benchmark must stratify by difficulty level; CS2 tasks are harder for all models |
| Test-case-based evaluation (Pass@k) is the accepted standard | Papers 23, 24 | Use execution-based evaluation; not BLEU score |
| Efficiency of generated code is not measured by existing CS1 benchmarks | Paper 6 | Gap we could address — add efficiency as a dimension to our benchmark |

### What Does NOT Work Yet (Open Problems)

| Open Problem | Evidence | What We Should Note |
|-------------|----------|---------------------|
| No model reliably matches human grading without oversight | Papers 1, 9, 10, 13 | Fully automated grading not solved; hybrid approach needed |
| LLM-generated tasks raise academic integrity risks | Papers 5, 16, 21 | Must address distinguishability and prompt hacking in our system design |
| Ceiling effect on MCQ-style benchmarks — models score similarly, no discrimination | Paper 4 | Avoid MCQ-only evaluation; use open-ended programming tasks |
| Efficiency of LLM-generated code consistently below human expert level | Paper 6 | Correctness alone is insufficient metric for educational code generation |
| Long/complex prompts degrade local model performance | Paper 1 | Prompt length must be controlled in our benchmark design |
| No benchmark aligns to actual CS1/CS2 curriculum progression | Papers 22, 23, 24 | Major gap — this is our research opportunity |
